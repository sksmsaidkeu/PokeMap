## 1. 공통 디자인 시스템 (Theme & Design System)

### 1.1 컬러 가이드 (Color Palette)

    * **회색 배경** (`#F0F0F0` 계열): 도감 목록 등 콘텐츠 집중도가 필요한 영역에 사용.

    * **메인 핑크**: 특정 기능 토글(Toggle) 시에만 강조색으로 활성화.
    * **레트로 레드**: 회원가입/로그인 및 도감 상세정보 탭의 몬스터볼 감성 포인트 컬러.
* **텍스트 컬러**: 
    * **글자**: Black (`#000000`)
    * **글자 테두리(Stroke)**: White (`#FFFFFF`) 느낌을 주어 가독성 확보.

## 2. 화면별 상세 UI 사양

### 2.1 회원가입 및 로그인 화면 (Sign Up / Sign In)
* **콘셉트**: 화이트 & 블랙 & 레드를 조합한 **몬스터볼(Pokéball)** 감성 디자인.
* **백그라운드 인터랙션**: 
    * PokeAPI의 다양한 아이템 스프라이트(몬스터볼, 도구 등)가 화면 위에서 아래로 비가 내리듯(Falling) 천천히 떨어지는 백그라운드 애니메이션 적용.

---

### 2.2 포켓몬 포획 탭 (Capture Page)
* **배경**: 
    * 상단: 하늘색 (`#cce8f4` 그라데이션)
    * 아랫부분: 사용자가 위치한 **'시(City)'의 대표 색상**으로 동적 매칭 설정.
* **레이아웃**: 세로 3단 배치 (중앙 정렬)

```
[ Player UI (라벨) ]
----------------------------------
 (포획 가능성)
     [ 포켓몬 이미지 영역 ]
     
----------------------------------
 [ No.0000 | 포켓몬 이름 ] (정보 바)
----------------------------------
 ◀ [ 포획하기 ] (액션 버튼)
```

1.  **데코레이션 및 메타 정보 (상단)**
    * **우측 상단**: `"Player UI"` 라벨 (리본/배너 형태의 말풍선 데코레이션).
    * **좌측 상단**: `"포획 가능성"` 태그 (포켓몬 이미지 바로 옆 위치). 
        * 포획 확률에 따라 `매우 낮음 / 낮음 / 보통 / 높음` 4단계로 텍스트 분기 표시.
2.  **상단 (메인 영역)**
    * 포켓몬 이미지/스프라이트가 중앙에 크게 배치되어 시선을 집중시킴.
3.  **중간 (정보 바)**
    * 가로로 긴 리본형 배너 (양끝이 화살표처럼 뾰족한 태그 형태).
    * 세로 구분선으로 2분할 구조: **[ 좌측: 도감 번호 ] | [ 우측: 포켓몬 이름 ]**
4.  **하단 (액션 버튼 - CTA)**
    * 좌측 끝이 뾰족한 화살표 리본 모양의 버튼.
    * 텍스트: `"포획하기"` (클릭 시 포획 시도 트리거).

### 2.3 포켓몬 도감 탭 (Pokedex Page)
도감 탭은 **[도감 목록]**과 **[상세 정보]** 두 가지 템플릿으로 구성됩니다.

#### ① 도감 목록 페이지 (List Template)
* **배경**: 연한 회색 (`#F0F0F0` 계열) 배경 위 흰색 카드형 컨테이너 레이아웃.
* **상단 헤더 영역**:
    * **좌측**: `< 뒤로가기` 버튼 (빨간색 원형 또는 둥근 사각형 배경).
    * **중앙**: `"검색 이름"` 텍스트 입력창 (포켓몬 이름 검색 인풋 필드).
    * **우측 버튼군**:
        1.  `🔍 (돋보기)` 아이콘 버튼: 검색 실행 버튼 (베이지/황토색 배경).
* **리스트 영역 (4열 그리드)**:
    * **개별 아이템 카드**: 흰색 배경 + 검은색 둥근 테두리(Rounded border)의 정사각형 카드.
    * **카드 내부**: 포켓몬 이미지 (가운데 정렬, 적절한 패딩 부여).
    * **카드 하단 텍스트** (좌측 정렬):
        * **1행**: `No.0000` 형태의 도감 번호 (회색의 작은 폰트).
        * **2행**: 포켓몬 한글 이름 (검은색 굵은 폰트, 조금 더 큰 사이즈).
* **그리드**: 4열 x N행 구성, 아이템 간 균등한 Gap 유지.

#### ② 포켓몬 상세 정보 탭 (Detail Template - Game Boy Style)
* **배경**: 보라색 격자무늬(체크보드 패턴) — 클래식 GBA(게임보이 어드밴스드) 오버월드 타일 감성.
* **최상단 바**: 푸른색 계열의 얇은 상태 바(Status Bar) 영역 배치.
* **콘텐츠 배치 (우측 정렬 배치된 3개의 독립형 박스 구조)**:

```
┌──────────────────────────────────────┐ [최상단 박스]
│ 🔴(몬스터볼 아이콘)  [빨간색 헤더]     │
├──────────────────────────────────────┤
│ 포켓몬 이름/번호/타입 기본 정보 표시    │
└──────────────────────────────────────┘
┌───────────────────┬──────────────────┐ [중간 박스]
│      키/몸무게    │       특성       │
└───────────────────┴──────────────────┘
┌──────────────────────────────────────┐ [하단 박스]
│█                                    █│ (좌우 빨간 캡슐형 사이드 바)
│           도감 설명 텍스트            │
│         (플레이버 텍스트 영역)        │
└──────────────────────────────────────┘
```

1.  **최상단 박스 (기본 정보 카드)**
    * 상단: 빨간색 배경 헤더 + 좌측에 원형 몬스터볼 아이콘(검은색/흰색 조합).
    * 하단: 흰색 영역으로 포켓몬 이름, 번호, 타입 등 기본 스펙 정보 표기.
2.  **중간 박스 (요약 정보 카드)**
    * 흰색 배경에 가로/세로 얇은 회색 구분선으로 나누어진 2단(혹은 다단) 그리드 구조.
    * 키, 몸무게, 특성 등 짧은 정보 요약 표시.
3.  **하단 박스 (도감 설명 카드 - 강조형)**
    * 좌측과 우측 끝에 빨간색 세로 바(캡슐형 사이드 액센트)가 들어간 강조 레이아웃.
    * 중앙의 넓은 흰색 영역에 도감 설명(플레이버 텍스트) 출력.

## 3. UI/UX 구현 인터랙션 핵심 요약 (개발 참고용)

* **토글 인터랙션**: 토글형 UI가 활성화되는 순간에만 **파스텔 핑크** 컬러를 사용하여 유저에게 직관적인 온/오프 피드백을 제공합니다.
* **그라데이션 연출**: 배경의 하늘색 그라데이션은 CSS `linear-gradient`를 사용하여 너무 무겁지 않고 맑고 경쾌한 톤을 유지합니다.
* **지역 연동 배경**: 포획 페이지 하단은 API 등으로 받아온 사용자의 현재 위치 정보(City)에 매핑된 대표 컬러가 자연스럽게 그라데이션으로 이어지도록 동적 스타일링(Dynamic Styling)을 적용합니다.
