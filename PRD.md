## 1. 제품 개요

PokeMap은 GPS API로 실제 행정구역(도/시)을 탐험하며 세부 권역별 포켓몬을 수집하는 '지역 확장형 포켓몬 RPG'다. 대한민국 8도 지도를 8개 포켓몬 지방에 매칭했고, 유저는 회원가입 시 GPS로 결정된 자기 위치에서 시작해 인접한 시(상/하/좌/우)로 이동하며 포켓몬과 조우, 포획해 도감을 채운다.

- **타깃**: 포켓몬 IP를 좋아하는 캐주얼 유저, 국내 지리에 익숙한 유저
- **플랫폼**: 반응형 웹(모바일 우선), Next.js App Router
- **핵심 재미**: "내가 사는 동네엔 어떤 포켓몬이 사는가"라는 로컬라이제이션 재미 + 강제 랜덤 인카운터의 긴장감 + 도(道) 정복형 진행감

## 2. 타깃 페르소나

**페르소나 A — "수집가 지민" (25세, 직장인)**
- 포켓몬 완주 경험 다수, 출퇴근길 자투리 시간에 짧게 플레이.
- 동기: 도감 완성, 자기 동네에 배정된 포켓몬 확인.
- 니즈: 세션이 짧아야 함(이동→강제 조우→3회 포획이 30초 이내), 도별 진행률이 한눈에 보여야 함.

**페르소나 B — "가챠러 도윤" (19세, 대학생)**
- 확률형 수집 게임 선호, 전설 포켓몬 등장 시 몰입도 급상승.
- 동기: 도감 100% 완공 후 전설 포획, "실패해도 확률이 영구히 오른다"는 손맛.
- 니즈: 전설 포획 실패가 완전한 손해가 아니라는 확신(§19 영구 누적 확률), 실패 연출이 좌절스럽지 않아야 함.

## 3. 성공 지표 (KPI)

| 지표 | 정의 | 목표(MVP 출시 4주 후) |
|---|---|---|
| D1 리텐션 | 가입 다음날 재방문 비율 | 35% 이상 |
| 평균 세션 이동 횟수 | 로그인당 시 이동 횟수 | 8회 이상 |
| 도 100% 완공률 | 하나 이상의 도를 100% 클리어(전설 출현 조건 충족)한 유저 비율 | 20% 이상 |
| 강제 조우 처리율 | 조우 발생 후 3회 시도까지 완주(포획/도망 확정)한 비율 | 95% 이상(이탈 최소화) |
| p95 API 응답시간 | `move-city`/`catch-attempt` | 300ms 이하 |

## 4. 게임 컨셉

- 한국 17개 광역자치단체("도") → 8개 포켓몬 지방으로 매칭 (`MapMatching.md` 원본):

| 포켓몬 지방 | 배정 도 |
|---|---|
| 관동지방 | 서울, 경기, 인천 |
| 성도지방 | 충북 |
| 호연지방 | 부산, 경남, 울산 |
| 신오지방 | 강원 |
| 하나지방 | 충남, 대전, 세종 |
| 칼로스지방 | 전북 |
| 알로라지방 | 광주, 전남, 제주 |
| 가라르지방 | 대구, 경북 |

- 각 도는 2~4개 "생활권"으로 세분화, 광역시는 생활권=도 자체. 각 생활권은 여러 시/군/구로 구성.
- 각 생활권은 도 공통 3종 + 생활권 고유 5종의 출현 포켓몬을 갖는다(`pokemonRare.md`). 전설 포켓몬은 도당 1마리 별도(§19).
- 예시(`pokemon.csv` 실제 시드): 서울특별시(관동지방, 공통) → 피카츄(BST 320), 리자몽(534), 이상해꽃(525). 서울 고유 → 거북왕(530), 이브이(325), 잠만보(540), 갸라도스(540). 서울의 전설은 뮤츠(BST 680).

## 5. 인증 및 온보딩

- **라우팅 가드**: 비인증 유저는 Login 페이지 외 어떤 라우트에도 접근 불가. URL 직접 입력으로 Map/Pokedex 진입 시도 시 자동으로 Login으로 리다이렉트.
- **로그인 페이지**: 유일한 비인증 접근 가능 페이지. 계정 생성/로그인 기능 제공.
- **GPS 온보딩**: 회원가입 완료 시점에 브라우저 Geolocation API로 좌표를 획득해 최초 시작 '도'와 그 안의 '생활권'을 자동 지정(`DB.md` `bootstrap-location` 함수). 위치 권한 거부 시 가입 자체를 막지 않고 기본 시작 도(서울특별시)로 폴백.

## 6. 플레이 루프

```
로그인/회원가입(GPS 온보딩) → [Map] 상/하/좌/우 인접 시 이동
            │
            ▼ (백엔드 강제 인카운터 판정)
       인카운터 성공? ──No──▶ Map 유지 (조우 없음)
            │Yes
            ▼ (강제 화면 전환, 사용자가 막을 수 없음)
       [Catch & Encounter 격리 탭] 조우 연출 → 최대 3회 포획 시도
            │
            ▼ (성공 또는 3회 소진)
       [Result] 팝업으로 성공/실패 표시
            │
            ▼ 페이드인/아웃 전환
       Map으로 복귀 (도 100% 달성 시 배너)

언제든: [Pokedex] 접근 가능 (헤더 또는 메뉴에서)
```

## 7. 공통 헤더 & 등급 시스템

Login 페이지와 Catch & Encounter 격리 탭을 **제외한 모든 페이지** 우측 상단에 고정 노출.

```
[등급 스프라이트] [트레이너 이름] ──────────────────── [⚙️ 설정]
   (등급명)
```

- 좌측: 등급 스프라이트 + 트레이너 이름, 스프라이트 바로 아래 작은 글씨로 등급명 표기.
- 등급(몬스터볼 → 슈퍼볼 → 하이퍼볼 → 마스터볼)은 전체 포켓몬 대비 유저 포획 비율로 실시간 자동 상승(§17).
- 우측: 설정 톱니바퀴 → 메뉴 → 로그아웃 버튼 → **2차 확인 모달**("정말 로그아웃 하실건가요 트레이너님?") → 확인 시 세션 만료 후 Login으로 리다이렉트, 취소 시 화면 유지.

## 8. 페이지별 상세 명세

### 8.1 Login
- 계정 생성/로그인, 최초 가입 시 GPS 온보딩(§5) 실행.
- 예외: 닉네임 중복 → 인라인 에러. 세션 만료 → Login으로 강제 리다이렉트.

### 8.2 Map
- `files/korea_map_data.json` 기반 지도, 시/군/구 폴리곤을 생활권 `color`로 채색.
- 플레이어 마커는 현재 위치한 시 위에 표시. 마커 기준 상/하/좌/우 4방향에 화살표 버튼이 오버레이되며, 지도에서 시를 직접 클릭해 이동하는 방식은 없다.
- 해당 방향에 인접 시가 없으면(그래프상 연결 없음) 그 화살표는 비활성/미표시 처리.
- **섬 지역 잠금**: 미해금 섬 지역(§20)에 속한 시 방향의 화살표만 비활성 처리(회색 + 잠금 아이콘). 그 외 육지 도는 잠금 없이 도 경계를 자유롭게 이동.
- 도가 100% 완공되면 그 도의 전설 출현지(상징적인 시, 예: 경기도의 수원시)가 지도에 특별 노출.
- **기본 줌**: 플레이어 마커 중심으로 확대해 플레이어와 인접 시들이 잘 보이는 수준으로 표시. 섬 지역 해금 등 특수 이벤트 시에만 일시적으로 전체 지도 줌아웃.
- 화살표 클릭(이동) → 그 방향의 인접 시로 `move-city` 호출 → 인카운터 성공 시 사용자 선택 없이 Catch & Encounter 탭으로 강제 전환.
- 예외: 네트워크 실패 시 토스트 + 이동 취소(서버 확정 후에만 마커 이동).

### 8.3 Catch & Encounter (격리 탭)
- **비공개 라우트**: URL 직접 입력이나 메뉴 클릭으로 진입 불가, 오직 유효한 강제 인카운터로만 진입.
- **일반 조우 연출**: 풀숲이 흔들리는 이펙트, 경쾌한 배틀 진입 효과음, 화면 셔터 연출.
- **전설 조우 연출**: 하늘이 어두워지고 번개/지진 이펙트, 긴장감 있는 전용 BGM, 전설 전용 화면 균열 연출.
- 최대 3회의 포획 시도(§15) 제공, 시도할 때마다 카운트 소모. 포획 성공 또는 3회 소진(도망) 시 종료.
- 종료 후 Result 팝업 노출, 이후 Map으로 페이드인/아웃 전환 복귀.
- 헤더 미노출 페이지(§7).
- 예외: 세션 만료(2분 미처리) 시 자동 도망 처리 후 Map 복귀.

### 8.4 Result
- 성공: 포획 이펙트 + "도감에 등록되었습니다", 신규 종이면 강조, 재포획이면 누적 횟수 표시.
- 실패(3회 소진): "도망쳤다" 연출.
- 도 100% 달성 조건을 이번 포획으로 충족했다면 별도 배너(전설 출현 알림).
- Result는 격리 탭 내부의 오버레이 팝업 — 별도 라우트가 아니라 Catch & Encounter 흐름의 마지막 상태.

### 8.5 Pokedex
- **도(Province) 순 정렬** — 8개 지방이 아니라 17개 도 순서로 그룹핑, 각 그룹 안에 그 도 배정 포켓몬 나열.
- 포획 성공(Unlocked): 컬러 일러스트 노출, 클릭/탭 시 상세 레이어(종족값, 타입, 도감 설명 텍스트) 팝업.
- 미포획(Locked): 명도 0% 검은 실루엣, 클릭/탭 인터랙션 완전 비활성화.
- 도별 완성률 프로그레스바(100% 기준과 함께 표시).
- 예외: 비로그인 접근 불가.

## 9. UI 흐름

```
Login ─┬─(신규가입)──▶ GPS 온보딩 ──▶ Map
        └─(기존)──────▶ Map
Map ──(인접 시 이동)──▶ [move-city 응답]
  ├─ 인카운터 없음 ──▶ Map 유지
  └─ 인카운터 발생(강제) ──▶ Catch & Encounter ──▶ Result(오버레이) ──▶ (fade) Map
Map / 헤더 ──▶ Pokedex ──(뒤로가기)──▶ Map
```

## 10. 상태 머신 (State Machine)

`encounter_sessions.status`가 단일 진실 소스(SSOT).

```
[NONE] --move-city(spawn=true 또는 전설 확정 조우)--> [PENDING]
[PENDING] --catch-attempt(성공, 1~3회차)--> [CAUGHT]
[PENDING] --catch-attempt 3회 모두 실패--> [FLED]
[PENDING] --expires_at 경과(2분)--> [FLED]
[CAUGHT] / [FLED] --Result 확인 후 fade--> [NONE]
```

- 클라이언트는 라우트 전환 시 항상 서버 `status`를 재확인. `PENDING`이 아닌데 격리 탭 진입 시도(뒤로가기 등) → Map으로 강제 리다이렉트.

## 11. 지역 시스템

- 계층: 포켓몬 지방(8, 표시 분류) > 도(17, 지역 구분 단위) > 생활권(N, 스폰 풀 단위) > 시(이동 최소 단위).
- 육지 도(제주도/울릉도·독도 제외)는 해금 개념이 없다 — 인접 시라면 도 경계와 무관하게 자유롭게 이동한다. 유일한 예외는 §20 최종 히든 지역(섬)으로, 내륙 전체 100% 클리어 조건이 그대로 유지된다.

## 12. 이동 시스템

- 이동 단위: 시. 플레이어 마커 기준 상/하/좌/우 화살표 버튼을 눌러 그 방향의 인접 시로 이동(지도 위 시 직접 클릭 이동 없음).
- 인접성은 지리적 경계 공유 그래프(`city_connections`)로 정의, 각 시의 `centroid` 대비 인접 시 중심 좌표의 상대 위치로 상/하/좌/우를 판정한다. 실시간 GPS 추적은 이동 자체에는 쓰이지 않는다(온보딩 시 1회만 사용, §5).
- 특정 방향에 인접 시 자체가 없거나, 그 시가 미해금 섬 지역(§20)에 속하면 그 방향 화살표는 비활성 처리되어 이동 명령을 낼 수 없다. 그 외 육지 도 방향은 항상 활성.
- 도 경계를 자유롭게 넘나든다(생활권 경계도 동일).
- 이동 1회 = 화살표 클릭 1회 = `move-city` 호출 1회 = 인카운터 판정 1회(전설 출현지 도착 시에는 판정 없이 확정 조우, §19).

## 13. 인카운터 시스템

- **강제 전환**: 인카운터 성공 시 지도에서 실루엣을 클릭하는 방식이 아니라, 이동 즉시 사용자 선택 없이 Catch & Encounter 격리 탭으로 화면이 강제 전환된다.
- 도착 생활권의 스폰 풀(공통 3 + 고유 5) 각 항목에 대해 독립 확률 판정(§14), 첫 성공을 스폰.
- 세션은 2분 TTL, 미처리 시 자동 도망 처리.

## 14. 스폰 알고리즘

- 확률은 종족값 합(BST)에 반비례, **5%~30%** 범위(`PokeMap_MainSystem.md` §3 / `pokemonRare.md` 원본 그대로).
- 계산: `spawn_rate = clamp(0.30 - (bst - 200) / (720 - 200) * 0.25, 0.05, 0.30)`.
- worked example: 피카츄(BST 320) → 약 24.2%. 잠만보(BST 540) → 약 13.7%.
- 전설은 이 판정 대상이 아니다(§19).

## 15. 포획 알고리즘

- 조우당 **최대 3회**의 독립 포획 시도, 회차 간 확률 변화 없음(볼 종류 선택 없음 — 몬스터볼~마스터볼은 §17의 유저 등급이지 아이템이 아니다).
- 확률은 종족값 반비례, **10%~90%** 범위: `catch_rate = clamp(0.90 - (bst - 200) / (720 - 200) * 0.80, 0.10, 0.90)`.
- worked example: 피카츄(320) → 약 83.8%. 잠만보(540) → 약 63.8%.
- Catch & Encounter 화면에는 원시 확률(%) 대신 4단계 "포획 가능성" 태그(매우 낮음/낮음/보통/높음)만 노출(`DESIGN.md` §2.2, `DB.md` §13.1). 클라이언트는 tier 값을 서버 응답 그대로 표시할 뿐 계산하지 않는다.
- 3회 중 첫 성공 시 즉시 확정, 3회 모두 실패하면 도망(재도전 불가, 이후 이동에서 같은 포켓몬이 다시 걸리길 기다려야 함).
- 전설 포켓몬은 별도 확률식 사용(§19).

## 16. 도감

- 종 단위(진화형 대표 1종만, `build_pokemon_db.py` 정책과 동일).
- **도(Province) 순 정렬**로 표시(§8.5). 최초 포획 시각/장소 기록, 이후 포획은 횟수만 누적.
- 상세 팝업에 종족값·타입·키·몸무게·대표 특성·도감 설명(flavor text) 노출(`DESIGN.md` §2.3②) — 이 중 flavor text·키·몸무게·특성은 현재 `pokemon.csv`에 없는 데이터로, 별도 수집이 필요하다(`DB.md` §4.7 데이터 갭).

## 17. 티어(등급) 시스템

주의: "티어"는 포켓몬의 희귀도 분류가 아니라 **유저의 전체 진행도 등급**이다.

- 4단계: 몬스터볼 → 슈퍼볼 → 하이퍼볼 → 마스터볼, 순서대로 상승.
- 산출 기준: 전체 포켓몬 도감 대비 유저의 포획 비율(도 구분 없이 전체 합산).
- 유저가 포켓몬을 잡을 때마다 실시간으로 재계산, 임계값 충족 시 즉시 등급 상승(원본 문서는 정확한 %를 명시하지 않음 — 잠정치: 몬스터볼 0%, 슈퍼볼 30%, 하이퍼볼 60%, 마스터볼 90%. 밸런스 확정 전까지 가정치임을 표기).
- 헤더(§7)에 등급 스프라이트 + 등급명으로 상시 노출.

## 18. 지역 해금

- 육지 도(17개 중 섬 지역 제외)는 포획률과 무관하게 처음부터 이동 제한이 없다 — 인접 시로만 갈 수 있다는 이동 규칙(§12)만 적용되고, "해금" 조건 자체가 없다.
- 유일한 해금 대상은 섬 지역(제주도/울릉도·독도, §20)이며 조건은 §20 그대로: 내륙 전체 도 100% 클리어.
- 판정 시점: 매 `move-city` 처리 후 `check_endgame_unlock` 재평가, 충족 시 즉시 해금(수동 확인 절차 없음).
- 섬 지역 해금 순간 Map은 일시적으로 전체 지도 줌아웃(§8.2)해 새로 열린 위치를 보여준 뒤, 다시 플레이어 기준 확대 상태로 복귀.

## 19. 전설 시스템 & 천장

전설 포켓몬은 도당 1마리이며, 별도의 일반-포켓몬용 천장 시스템은 존재하지 않는다 — 아래 전설 전용 메커니즘 자체가 "천장(pity)" 역할을 한다.

- **등장 조건**: 현재 도의 일반 포켓몬을 **100% 포획(도감 완공)**하면, 그 도의 수호신 격인 전설 포켓몬 위치가 지도에 노출된다.
- **위치**: 도 안에서 가장 상징적인 시(예: 경기도의 수원시)에 고정 배치.
- **조우 방식**: 강제 랜덤 인카운터가 아니라, 그 시로 직접 이동하면 확률 판정 없이 확정 조우한다.
- **포획 확률**: 기본 **3%**로 시작. 실패 시 그 방문의 3회 시도가 모두 소진된 것으로 간주, 도망 처리 후 **1시간 쿨타임**이 걸린다. 쿨타임이 지나 재도전할 때마다 확률이 기본값에서 **영구히 1%p씩 누적 증가**한다(1회 실패 후 4%, 2회 실패 후 5% …). 리셋 없음.
- 포획 성공 시 해당 도의 전설 관련 재도전 로직은 더 이상 발생하지 않는다.

## 20. 최종 히든 지역

- **제주도 / 울릉도·독도**: 다른 도와 달리 §18의 자유 이동 대상이 아니다 — 인접한 육지 도가 없어 도달 자체가 별도 해금 조건에 걸린 유일한 예외.
- **해금 조건**: 내륙의 모든 도(경기, 강원, 충청, 전라, 경상 등 섬 지역을 제외한 전체)를 각각 100% 클리어(일반 포켓몬 전량 포획)해야 잠금 해제.
- 해금 전에는 지도에서 선택/이동 자체가 불가능하다.
- **데이터 갭**: 제주도는 이미 알로라지방 소속으로 시드되어 있으나(`MapMatching.md`, `pokemon.csv`), 울릉도·독도는 `korea_living_areas.csv`/`pokemon.csv` 어디에도 존재하지 않는다 — 별도 도/생활권/시/포켓몬 배정 데이터가 추가로 필요하다(`DB.md` §16).

## 21. 사운드

- 지방별 배경음 1종(이동 중 재생).

## 22. 애니메이션

- 일반 조우: 풀숲 흔들림 이펙트 + 화면 셔터.
- 전설 조우: 하늘이 어두워짐 + 번개/지진 이펙트 + 화면 균열.
- 격리 탭 → Map 복귀: 부드러운 페이드인/페이드아웃 트랜지션(화면이 툭 끊기지 않게).
- `prefers-reduced-motion` 설정 시 화면 셔터/균열/파티클 생략, 페이드만 유지.

## 23. 접근성 & 반응형

- 모바일 우선 브레이크포인트: 360px(최소), 768px(태블릿), 1024px 이상(데스크톱, 지도 좌측 + 사이드패널).
- 색맹 유저를 위해 생활권 색상 구분에 아이콘/텍스트 병기.
- 터치 타깃 최소 44×44px.
- GPS 권한 거부 유저를 위한 폴백 시작 위치 제공(§5).
